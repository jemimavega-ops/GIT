1.- git add "nombre del archivo" --> agregar cambios del archivo NO USAR COMILLAS

2.- git status --> ver estado de los archivos
    - Verde : estado guardado
    - Rojo : No guardado

3.- git commit -m"comentario" --> para cambiar a estado de commit

4.- git remote add origin "http del repositorio de github" --> enlazar repositorio del pc con el de github
    4.1.- git push origin master --> agregar cambios desde el repositorio del pc al repositorio de github
    4.2.- git remote -v --> ver si se enlazó bien el repositorio del pc con el de github

5.- Si editas en el github y transfieres esa edición al archivo del pc:
pasos:
    1.- git add <nombre del archivo> --> prepara el archivo para cambios
    2.- git commit -m <comentario del cambio>
    3.- git pull origin master --> muestra por fin los cambios
    4.- git add <nombre del archivo> --> para confirmar los cambios y guardarlos (no funciona el ctrl S)

6.- git push --set-upstream origin master --> volver a enlazar con la rama virtual
    tambien sirve para guardar cambios desde el archivo en terminal al virtual cuando el git push origin master no funciona
    pasos para eto:
    1.- git add <nombre del archivo> --> prepara el archivo para cambios
    2.- git commit -m <comentario del cambio>
    3.- git push --set-upstream origin master --> muestra por fin los cambios
